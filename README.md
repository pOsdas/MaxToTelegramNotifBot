# MAX → Telegram Notifier

Сервис держит **один Chromium открытым постоянно**, использует один постоянный профиль MAX и раз в час проверяет наличие непрочитанных сообщений. Браузер не запускается заново перед каждой проверкой.

После перезапуска контейнера Chromium запускается снова, но использует тот же каталог `runtime/browser-profile`, поэтому cookies и авторизация сохраняются.

## Архитектура

```text
MAX Web в постоянном Chromium
            ↓
проверка DOM раз в 3600 секунд
            ↓
сравнение с последним состоянием в SQLite
            ↓
уведомление через Telegram Bot API
```

В проекте нет VPN, прокси, PostgreSQL, Redis и Nginx. Сервер в Нидерландах напрямую подключается к MAX и Telegram. Входящие порты 80 и 443 не требуются.

noVNC опубликован только на `127.0.0.1:6080` сервера и доступен через SSH-туннель. Порт 6080 не открывается во внешний интернет.

## Важное ограничение

Официальный API MAX предназначен для ботов и не предоставляет доступ к личным сообщениям обычного аккаунта. Поэтому здесь используется автоматизация веб-интерфейса личного аккаунта. После изменения разметки MAX алгоритм поиска непрочитанных сообщений может потребовать корректировки селекторов.

Первая версия использует несколько независимых признаков: число в заголовке вкладки, `aria-label`, `data-testid`, `data-qa`, а также классы с `unread`, `badge` и `counter`. При ошибке сохраняются скриншот, HTML и метаданные страницы.

## 1. Загрузка проекта на сервер

Распакуй архив и перейди в каталог проекта:

```bash
cd /opt
unzip max-telegram-notifier.zip
cd max-telegram-notifier
```

Либо сначала распакуй проект на Windows, создай репозиторий Git и клонируй его на сервер.

## 2. Создание `.env`

```bash
cp .env.example .env
nano .env
```

Обязательные значения:

```env
TELEGRAM_BOT_TOKEN=
TELEGRAM_CHAT_ID=
```

Токен создаётся через `@BotFather` в Telegram.

### Как получить `TELEGRAM_CHAT_ID`

1. Создай бота через `@BotFather`.
2. Вставь токен в `.env`, оставив `TELEGRAM_CHAT_ID` пустым.
3. Отправь созданному боту сообщение `/start`.
4. Выполни:

```bash
docker compose build
docker compose run --rm notifier python scripts/get_telegram_chat_id.py
```

Скрипт выведет готовую строку вида:

```text
TELEGRAM_CHAT_ID=123456789
```

Вставь полученное число в `.env`.

## 3. Проверка Telegram

```bash
docker compose run --rm notifier python scripts/send_test_notification.py
```

В Telegram должно прийти тестовое сообщение.

## 4. Запуск

```bash
docker compose up -d --build
```

Проверь состояние:

```bash
docker compose ps
docker compose logs -f --tail=200 notifier
```

## 5. Вход в MAX через noVNC

На Windows открой отдельное окно CMD или PowerShell и оставь команду работающей:

```powershell
$ServerIp = Read-Host "Введите IP сервера"
ssh -L 6080:127.0.0.1:6080 "root@$ServerIp"
```

Затем открой в браузере:

```text
http://127.0.0.1:6080/vnc.html?autoconnect=true&resize=scale
```

В окне noVNC будет открыт Chromium с веб-версией MAX.

1. Войди в свой аккаунт MAX по номеру телефона и коду.
2. Дождись появления списка чатов.
3. Не закрывай вкладку MAX.
4. SSH-туннель после входа можно закрыть. Контейнер и Chromium продолжат работать на сервере.

## 6. Как проходит проверка

- Chromium запускается один раз при старте контейнера.
- Вкладка MAX остаётся открытой.
- Каждые `3600` секунд Python читает текущее состояние вкладки. Пока вход в MAX не выполнен или произошла ошибка, повторная попытка делается через 60 секунд.
- Если непрочитанных стало больше или изменился чат/фрагмент сообщения при том же количестве, отправляется уведомление.
- Если количество уменьшилось, уведомление не отправляется.
- Последнее состояние хранится в SQLite и переживает перезапуск контейнера.
- При временной ошибке Telegram уведомление остаётся в базе и повторяется позже.

Интервал задаётся в `.env`:

```env
MAX_CHECK_INTERVAL_SECONDS=3600
MAX_RETRY_INTERVAL_SECONDS=60
```

## 8. Диагностика MAX

При ошибках файлы автоматически появляются в:

```text
runtime/screenshots/
```

Там сохраняются:

- PNG-скриншот;
- HTML страницы;
- JSON с адресом, заголовком и причиной сохранения.

Ручная диагностика:

```bash
make inspect
```

Команда временно остановит основной контейнер, запустит Chromium с тем же профилем, сохранит диагностику и снова запустит сервис.

## 9. Проверка памяти

```bash
docker stats max-telegram-notifier
free -h
```

Для Chromium контейнеру выделен `/dev/shm` размером 1 ГБ. На сервере уже есть zram 1,9 ГБ, поэтому дополнительный swap-файл сейчас не нужен.

## 10. Безопасность

- noVNC слушает только localhost сервера.
- Для доступа используется SSH-туннель.
- Токен Telegram хранится только в `.env` и не попадает в Git или Docker-образ.
- Каталог профиля MAX не попадает в Git и ZIP после начала работы.
- Порты PostgreSQL и Redis отсутствуют.
- VPN и прокси не используются.

После первого успешного запуска рекомендуется заменить вход по паролю root на SSH-ключ и отдельного пользователя для развёртывания.
